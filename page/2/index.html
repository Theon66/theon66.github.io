Maven学习

Posted on 2018-12-09

前言

虽然我一直都在用Maven构建项目，但是真实情况是，对Maven的了解只有一点皮毛，基本上除了除了依赖、clean、install就什么都不知道了。使用过程中经常有一些错误的使用方式，离发挥Maven的最大威力还有很远的差距。所以决定参考《Maven实战》来做一个系统的学习。

为什么用Maven

构建：编译、运行单元测试、生成文档、打包和部署。

自动化构建，抽象构建过程，提供构建任务实现（自己编写实现插件）

什么是Maven

Maven不仅仅是构建工具，还是一个依赖管理工具和项目信息管理工具。它提供了中央仓库，能帮助我们自动下载构建。

初识Docker

Posted on 2018-11-28 Edited on 2018-12-02

认识容器

容器是一种标准化的软件单元

有了容器，就可以将软件运行所需的所有资源打包到一个隔离的容器中。容器与虚拟机不同，不需要捆绑一整套操作系统，只需要软件工作所需的库资源和设置。系统因此而变得高效、轻量、自给自足，还能保证部署在任何环境中的软件都能始终如一地运行。

将软件打包成标准化单元，以用于开发、交付和部署

容器镜像是轻量的、可执行的独立软件包，包含软件运行所需的所有内容：代码、运行时环境、系统工具、系统库和设置。容器化软件适用于基于 Linux 和 Windows 的应用，在任何环境中都能够始终如一地运行。容器赋予了软件独立性，使其免受外在环境差异（例如，开发和预演环境的差异）的影响，从而有助于减少团队间在相同基础设施上运行不同软件时的冲突。

什么是Docker

Docker 是世界领先的软件容器平台。开发人员利用 Docker 可以消除协作编码时“在我的机器上可正常工作”的问题。运维人员利用 Docker 可以在隔离容器中并行运行和管理应用，获得更好的计算密度。企业利用 Docker 可以构建敏捷的软件交付管道，以更快的速度、更高的安全性和可靠的信誉为 Linux 和 Windows Server 应用发布新功能。

为什么使用Docker

抄三段官网首页的话

摆脱应用依赖矩阵
消除“在我的机器上可正常工作”的问题，一劳永逸。把应用和其依赖关系一起打包在 Docker 容器中，以便在开发、测试和部署过程中实现可移植性和可预测性。

适用于任何技术栈
可在任何环境中部署微服务和传统应用，而无需费时费力地重新编码。用容器来隔离应用，消除冲突，加强安全性。

更好的团队协作
简化开发人员和运维人员间的协作，让各项功能和修复更快地应用到生产环境中。高效的开发人员可以频繁地发布应用，激发创造力，构建绝佳软件。

开发人员使用Docker

Docker 能够自动执行重复性任务，例如搭建和配置开发环境，从而解放了开发人员以便他们专注在真正重要的事情上：构建杰出的软件。
开发人员使用了 Docker，就不必安装和配置复杂的数据库，也无需在不兼容语言工具链版本之间切换时担心。应用容器化之后，其复杂性就被转移到能够轻松构建、共享和运行的容器中。当有新同事安排到新的代码库时，无需再费时费力地安装软件和解释设置过程。以 Dockerfile 文件形式发布的代码使用起来非常简单：首先安装 Docker 和编辑器，然后拉取作为 Docker 镜像打包在一起的依赖资源，短短数分钟，就能够构建和调试应用了。

门户网站优化记录

Posted on 2018-11-01 Edited on 2018-11-02

背景

最近优化了公司的门户网站，由于首页十分臃肿，有大量的图片和Echart地图的内容。从一开始完全加载完毕需要2分钟以上，优化到只要7s左右，记录一下都用了哪些方法来优化网站。

处理图片

压缩图片是最直接也是最有效的方法，在带宽不变的情况下，资源越大，网站加载时间也就越长。而未经处理和压缩的图片经常大的吓人。

  1. 把PNG的图片转换为jpg通常可以缩小很多体积。（这里我不太确定jpg是不是就是最理想的图片类型）
  2. 使用nginx图片过滤模块，重置图片尺寸，这样可以保证网页上图片显示的实际大小与请求的图片大小相同，如果请求的图片尺寸太大（大于页面上显示），则等于浪费了带宽。还可以避免存储多个尺寸的同一张图片。要注意的是此方法不适合并大量较大的网站，因为nginx在处理图片时会消耗服务器资源，量过大的时候会增加服务器负担。
  3. 使用懒加载，用了一个叫lozad的js插件，只加载可视范围内的图片。

压缩js、css等静态资源

  • 启用nginx的Gzip功能，可以大幅压缩静态文件的体积，减少网页传输的时间。

提高带宽

这是最简单粗暴有效的办法，有钱就行。

使用jsp/静态页面/ajax？

  • ajax： 网站本来大部分内容都是使用ajax加载的，个人认为ajax并不适合门户网站这种变动较小的网站，而且使用ajax无法使用lozad进行懒加载，而且ajax加载的内容无法被百度等搜索引擎所抓取，导致用户搜索不到。
  • jsp：使用传统的mvc模式，我这次就是用的jsp，因为改起来比较容易，而且没有ajax上面所说的缺点。
  • 静态页面：在每次后台上传新的文章后，使用模板引擎生成静态页面，用户直接访问静态页面。显然这是最好的办法，更快的访问速度，使用服务器资源最少。

合并多个js，css文件

理论上来说，请求多个js文件所花时间会大于单个js，但我实际合并过后，不知为何网站加载时间没有缩短，反而还稍微有增加。但从几个测速网站来看，单个js确实好于多个js。这里的原因暂时还不清楚。

一些其他有用的东西

  1. Google Pagespeed Insights, 这个谷歌产品可以帮你分析网站，给你优化网站的建议。
  2. Pagespeed，此工具可以集成到apache或nginx中，可以自动帮你进行网站优化，据说效果非常好，虽然我自己没有用，因为我已经手动做了大部分的工作。但这个东西有一个缺点，似乎会占用不少的服务器cpu资源。
