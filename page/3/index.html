RabbitMQ 基本概念

Posted on 2018-09-26 Edited on 2019-01-15

是什么

消息代理。类似一个邮局。

为什么要用（使用场景）

  • 消息的主动推送，系统A发送消息，系统B监听并接受消息，达到了系统A向系统B主动推动消息的目的。

  • 异步处理，主要目的是减少请求响应时间和解耦。所以主要的使用场景就是将比较耗时而且不需要即时（同步）返回结果的操作作为消息放入消息队列。同时由于使用了消息队列，只要保证消息格式不变，消息的发送方和接收方并不需要彼此联系，也不需要受对方的影响，即解耦和。

    术语

  • 生产者：发送消息的程序

  • 队列：类似于邮箱。虽然消息通过 RabbitMQ 在你的应用中传递，但是它们只能存储在队列中。队列只受主机的内存和磁盘限制的限制，它本质上是一个大的消息缓冲区。不同的生产者可以通过同一个队列发送消息，此外，不同的消费者也可以从同一个队列上接收消息。

  • 消费者(consumer)：等待接受消息的程序。

  • 交换器(Exchanges): 交换器是非常简单的。它一边收到来自生产者的消息，另一边将它们推送到队列。交换器必须准确知道接收到的消息如何处理。是否被添加到一个特定的队列吗？是否应该追加到多个队列？或者是否应该被丢弃？这些规则通过交换器类型进行定义。

  • 临时队列(Temporary queues): 在 Java 客户端中，我们可以使用 queueDeclare() 方法来创建一个非持久的、唯一的、自动删除的队列，且队列名称由服务器随机产生。

  • “发布/订阅”模式：提供一个信息给多个消费者。

  • 绑定（Bindings）: 交换器和队列之间的关系称为绑定。

补充

交换器的四种类型

  1. fanout 类型的交换器非常简单，它只是将所有收到的消息广播到所有它所知道的队列。

有什么用

首先引用两句阿里手册里关于枚举的话

【强制】二方库里可以定义枚举类型，参数可以使用枚举类型，但是接口返回值不允许使用枚 举类型或者包含枚举类型的 POJO 对象

【推荐】如果变量值仅在一个范围内变化，且带有名称之外的延伸属性，定义为枚举类。下面 正例中的数字就是延伸信息，表示星期几。 正例：public Enum { MONDAY(1), TUESDAY(2), WEDNESDAY(3), THURSDAY(4), FRIDAY(5), SATURDAY(6), SUNDAY(7);}

总结一下,枚举只有两个作用

  1. 当作参数
  2. 如果变量值仅在一个范围内变化，且带有名称之外的延伸属性，定义为枚举类

解释一下processData和contentType

processData (默认: true)
类型: Boolean
默认情况下，通过data选项传递进来的数据，如果是一个对象(技术上讲只要不是字符串)，都会处理转化成一个查询字符串，以配合默认内容类型 “application/x-www-form-urlencoded”。如果要发送 DOM 树信息或其它不希望转换的信息，请设置为 false。
contentType (默认: ‘application/x-www-form-urlencoded; charset=UTF-8’)
类型: String
发送信息至服务器时内容编码类型。默认值是”application/x-www-form-urlencoded; charset=UTF-8”，适合大多数情况。如果你明确地传递了一个内容类型（Content-Type）给 $.ajax()，那么他必定会发送给服务器（即使没有数据要发送）。数据将总是使用UTF-8字符集传递给服务器；你必须在服务器端进行适当的解码。

AOP实例：日志记录

Posted on 2018-08-14 Edited on 2018-08-15 In AOP

背景

公司要求记录报表里用户的操作，以前没有使用过AOP，一开始想着编写几个接口，在要记录的地方调一下接口。其实这种方法很蠢，AOP正是为了解决这种问题而生的。

思路

最开始采用自定义注解的方式，即在要记录的方法处加上一句注解。但其实这是一种糟糕的设计，这样会把这些业务模块的关注点变的不单一，业务模块不需要知道自己会不会被记录。

所以采用AspectJ表达式才是正确的姿势。
